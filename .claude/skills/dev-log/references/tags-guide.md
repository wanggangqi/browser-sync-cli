# 标签规范指南

> 标签用于快速分类和语义检索，统一的标签体系有助于提高查询效率。

---

## 标签体系

### 核心业务标签

| 标签 | 说明 | 使用场景 |
|------|------|----------|
| `# 用户` | 用户相关 | 用户认证、用户信息、用户管理 |
| `# 积分` | 积分系统 | 积分获取、积分消耗、积分流水 |
| `# 商品` | 商品管理 | 商品 CRUD、上下架、库存 |
| `# 订单` | 订单系统 | 订单创建、支付、状态流转 |
| `# 支付` | 支付相关 | 微信支付、支付回调、退款 |
| `# 兑换` | 积分兑换 | 兑换商品、兑换规则 |

## 标签使用规则

### 1. 标签数量

每条日志建议打 **2-5 个标签**：
- 至少 1 个业务标签（如 `# 积分`、`# 订单`）
- 可选 1 个技术标签（如 `# 数据库`、`# 接口`）
- 可选 1 个流程标签（如 `# Bug 修复`、`# 需求`）

### 2. 标签格式

- 使用 `# 标签名` 格式
- 标签名中不含空格
- 中文标签使用汉字，不使用拼音

### 3. 标签优先级

优先使用上表中定义的**标准标签**，如确实需要可临时创建新标签，但应避免：
- 含义重复的标签（如 `# 支付` 和 `# 付款`）
- 过于细粒度的标签（如 `# 订单创建 ` 不如`# 订单`）
- 个人化的标签（应团队统一）

## 标签检索技巧

使用语义检索时，可以结合标签进行过滤：

```sql
-- 查询所有积分相关的变更
SELECT * FROM dev_logs WHERE tags && ARRAY['积分'];

-- 查询积分相关的 Bug 修复
SELECT * FROM dev_logs
WHERE tags && ARRAY['积分', 'Bug 修复'];

-- 查询某时间段内的安全相关变更
SELECT * FROM dev_logs
WHERE tags && ARRAY['安全']
AND log_date BETWEEN '2026-03-01' AND '2026-03-31';
```

## 标签维护

定期审查标签使用情况，合并重复标签，淘汰不使用标签。

建议每季度执行一次标签审计：

```sql
-- 查看所有标签及其使用次数
SELECT unnest(tags) as tag, count(*) as usage_count
FROM dev_logs
GROUP BY tag
ORDER BY usage_count DESC;
```
